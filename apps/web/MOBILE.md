# Suzi Chat — Mobile App (Capacitor) Handoff

This wraps the existing Next.js web app as native **Android** and **iOS** apps using
[Capacitor](https://capacitorjs.com/). The web UI is built as a static export and
bundled inside each native app; it talks to the production backend over HTTPS.

- **App name:** Suzi Chat
- **App / bundle id:** `com.suzichat.app` — change this if your store accounts use a
  different id (see "Changing the bundle id" below).
- **Backend API:** `https://suzichat.com/api` (baked in at build time).
- **Strategy:** static export (`output: "export"`) bundled into the app. Content is
  shipped with the binary, so a new store build is required to ship UI changes.

---

## 1. Prerequisites

| Target  | Needs |
|---------|-------|
| Both    | Node.js ≥ 20, pnpm ≥ 10 (`npm i -g pnpm`) |
| Android | Android Studio (latest) + JDK 17 |
| iOS     | macOS + Xcode 15+ (CocoaPods is **not** needed — uses Swift Package Manager) |

## 2. Build the web bundle and sync it into the native projects

From the **repo root**:

```bash
pnpm install
pnpm --filter web cap:sync
```

`cap:sync` runs `build:mobile` (static export pointing at `https://suzichat.com/api`)
and then `cap sync` (copies the bundle + plugins into `android/` and `ios/`).

> The web bundle inside the native projects is git-ignored and regenerated by this
> step, so **always run `cap:sync` after a fresh clone** before building natively.

To change the API URL, edit the `build:mobile` / `cap:sync` scripts in
`apps/web/package.json` (`NEXT_PUBLIC_API_BASE_URL=...`).

## 3. Build & publish — Android

```bash
pnpm --filter web exec cap open android
```

In Android Studio: let Gradle sync, then **Build > Generate Signed Bundle / APK**
→ choose **Android App Bundle (.aab)**, create/select your keystore, build `release`.
Upload the `.aab` to Google Play Console.

## 4. Build & publish — iOS (macOS only)

```bash
pnpm --filter web exec cap open ios
```

In Xcode: select the **App** target → **Signing & Capabilities** → set your Team and a
unique bundle id. Then **Product > Archive** → distribute to App Store Connect.

---

## 5. ⚠️ Required server-side change (CORS)

The native app's web origin is **not** `suzichat.com` — it is `https://localhost`
(Android) and `capacitor://localhost` (iOS). The production API **and** the Socket.IO
gateway at `suzichat.com` must allow these origins, or login, data, and realtime
(chat/calls/live) will fail.

Add to the API's allowed origins (e.g. `CORS_ORIGINS`) and the Socket.IO CORS config:

```
https://localhost
capacitor://localhost
```

This is a backend/deploy change on the server that hosts `suzichat.com`.

## 6. Permissions (already configured)

- **Android** (`android/app/src/main/AndroidManifest.xml`): `INTERNET`, `CAMERA`,
  `RECORD_AUDIO`, `MODIFY_AUDIO_SETTINGS`.
- **iOS** (`ios/App/App/Info.plist`): `NSCameraUsageDescription`,
  `NSMicrophoneUsageDescription`, `NSPhotoLibraryUsageDescription`.

These cover voice/video calls, going live, voice messages, and media uploads.

## 7. App icons & splash screen (TODO — apply your branding)

The projects currently use Capacitor's default icons. To generate branded assets,
drop a 1024×1024 PNG at `apps/web/assets/icon.png` (and optional `splash.png`) and run:

```bash
pnpm --filter web add -D @capacitor/assets
pnpm --filter web exec capacitor-assets generate --iconBackgroundColor "#160b39"
```

Then re-run `pnpm --filter web cap:sync`.

## 8. Changing the bundle id

If you must use a different id than `com.suzichat.app`:
1. Edit `appId` in `apps/web/capacitor.config.ts`.
2. Android: update `applicationId` in `android/app/build.gradle` and the
   `package_name` / `custom_url_scheme` in `android/app/src/main/res/values/strings.xml`.
3. iOS: set the bundle id in Xcode signing.
4. Run `pnpm --filter web cap:sync`.

## 9. Notes for reviewers / maintenance

- **Routing:** static export can't serve arbitrary dynamic URLs, so user-generated
  screens use query params: room chat `/app/rooms/view?r=<slug>`, public profile
  `/app/profile/view?u=<username>` or `?uid=<id>`, game session
  `/app/games/<gameId>/session?s=<sessionId>`.
- **Store assets needed:** app icon, screenshots, description, and a privacy policy URL
  (`https://suzichat.com/privacy`) / terms (`https://suzichat.com/terms`).
- To ship a UI update: rebuild (`pnpm --filter web cap:sync`) and submit a new binary.
